Machine virtuelle

Afin de tester Feel++, Cemosis vous propose une marche à suivre pour installer (ou télécharger) une machine virtuelle dont l'utilisation est décrite en fin de page.

Marche à suivre pour construire sa propre machine

Nous recommandons l'installation de Feel++ sur debian unstable (Nous recommandons d'utiliser le fichier netinst.iso). Vous pouvez installer Feel++ soit dans une machine physique, soit virtuelle (Virtual Box).

Une fois la machine installée, nous vous conseillons le script suivant (en root) afin de satisfaire toutes les dépendances de Feel++ :

apt-get update
apt-get upgrade
apt-get install feel++-doc libfeel++-dbg  libfeel++-dev libfeel++1 git gmsh libgmsh-dev

Une fois ces paquets installés, vous avez les exemples de la documentation installés dans votre système! Si vous désirez tester directement les sources, voici la marche à suivre (utilisateur lambda) :

cd $HOME
git clone https://github.com/feelpp/feelpp.git
mkdir buildFeel; cd buildFeel
cmake ~/feelpp -DCMAKE_CXX_COMPILER=`which clang++`
make -j4

Utiliser la machine virtuelle de Cemosis

La machine virtuelle de Cemosis n'est actuellement pas disponible.


Nous mettons à votre disposition une machine virtuelle (Debian unstable) mise à jour par quinzaine afin de vous permettre de tester Feel++ rapidement.

Veuillez indiquer ci-dessous votre adresse mail afin de recevoir les indications de téléchargement de la machine (plusieurs GB).
